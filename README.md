# mcp-latex-proofread

Want an LLM to proofread your LaTeX draft in detail, but not "helpfully" rewrite the text while it is at it?
`mcp-latex-proofread` exists for exactly that use case.

## Key features

- Enforces that only managed `\llm{...}` comments can be added, edited, or removed
- Verifies every write before it reaches disk
- Lets the agent revisit the same comment later via a stable `llm:id`
- Works naturally inside a normal LaTeX author-todo workflow

Managed comments look like this:

```latex
\llm{The inequality sign appears to be reversed here.}% llm:id=a1b2c3d4
```

The `llm:id` is generated by the server and lets the agent edit or remove the same comment later.

## Install with uv

Before continuing, make sure to have [uv](https://docs.astral.sh/uv/) installed.
You do not need to clone this repository.

## Claude Code

From your LaTeX project directory, run:

```bash
claude mcp add mcp-latex-proofread -- \
  uvx git+https://github.com/holgerdell/mcp-latex-proofread mcp-latex-proofread
```

This registers the MCP server globally for the current user.
You can also edit `.claude/settings.json` manually if you prefer.

Then use Claude Code in a LaTeX project directory and ask for operations such as:

- `Insert an \llm comment after the introduction heading in paper.tex.`
- `List all managed \llm comments in chapter2.tex.`
- `Replace the comment with id abc12345 in notes.tex.`

## OpenCode

From your LaTeX project directory, run:

```bash
opencode mcp add
```

When prompted:

1. Choose a local MCP server.
2. Name it `mcp-latex-proofread`.
3. Use `uvx` as the command.
4. Use `git+https://github.com/holgerdell/mcp-latex-proofread`, `mcp-latex-proofread` as the arguments.

You can also add the same MCP server manually in `~/.config/opencode/opencode.json` if you prefer.

Then start OpenCode in that same LaTeX project directory and give the same kinds of requests.

## What The Server Can Do

- Read line ranges from a `.tex` file for context
- List all managed `\llm` comments and their IDs
- Insert a new managed comment after a line number or text match
- Replace or remove an existing managed comment by ID
- Validate that all managed comments in a file are well formed

## What The Server Can't Do

This server only constrains what happens through this MCP interface. If your coding agent also has direct file-editing tools, shell access, or broad write permissions, it may still modify the LaTeX source by other means. Choosing the right permissions and approval settings for your agent is still up to you.

## Recommended LaTeX Setup

The example file uses visible draft comments, the likes of which you may have already set up with your co-authors anyway:

```latex
\usepackage[usenames,dvipsnames]{xcolor}
\usepackage{xparse}
\NewDocumentCommand{\todo}{ O{Fuchsia} O{to-do} +m
}{\upshape{\color{#1}{\Large[}{\sffamily\bfseries #2:} #3{\Large]}}}

\newcommand{\authorA}[1]{\todo[RoyalPurple][A]{#1}}
\newcommand{\authorB}[1]{\todo[OliveGreen][B]{#1}}
\newcommand{\authorC}[1]{\todo[RoyalBlue][C]{#1}}
\newcommand{\authorD}[1]{\todo[OrangeRed][D]{#1}}
\newcommand{\authorE}[1]{\todo[DarkOrchid][E]{#1}}

\NewDocumentCommand{\llm}{ o m }{%
  \todo[RoyalPurple][\IfValueTF{#1}{#1}{LLM}]{#2}%
}
```

That makes `\llm{...}` notes visible in the compiled PDF and lets them live alongside a normal author todo workflow. The optional argument is intended for the model name, for example `\llm[gpt-5.4]{...}` or `\llm[opus-4.6]{...}`, instead of the default `LLM` tag.

If you prefer comments to compile away completely, you can instead use:

```latex
\newcommand{\llm}[1]{}
```
